## 1. 개요 (Overview)

### 1.1 문서 목적

GDC 2026 행사 기간 중 NEXUS 부스를 방문하는 관람객을 대상으로 미션 달성 여부에 따라 $CROSS 토큰을 배포하고, 해당 배포 과정에서 발생 가능한 어뷰징을 방지하기 위한 관리 시스템의 PRD이다.

### 1.2 행사 개요

| 항목 | 내용 |
|------|------|
| 행사명 | GDC 2026 (Game Developers Conference) |
| 일정 | 2026.03.09 ~ 03.13 (5일간, 전시/세션 동시 진행) |
| 장소 | Moscone Center, San Francisco, CA |
| 참여 주체 | NEXUS, Verse8, 전략 파트너사 |
| 부스 컨셉 | "From Idea to Live Game" — End-to-End 실시간 시연 |
| 토큰 배포 대상 기간 | 행사 3일간 (구체 일정 별도 확정) |

### 1.3 배경 및 목표

NEXUS는 GDC 2026에서 CROSS 생태계의 핵심 메시지인 "Game × AI × Blockchain"을 관람객에게 체험시키고자 한다. 미션 완수 관람객에게 $CROSS 토큰을 보상으로 제공함으로써 CROSS 생태계 참여를 유도하고, 동시에 토큰 배포 과정의 투명성과 건전성을 담보하는 관리 체계를 구축한다.

- CROSS 생태계 체험 확대 및 인지도 제고
- 미션 기반 토큰 배포를 통한 관람객 인게이지먼트 강화
- 어뷰징 방지 및 투명한 토큰 전송 기록 관리
- 일별 정산 및 검증 체계를 통한 내부 통제 강화

## 2. 미션 및 보상 구조

### 2.1 미션 구성

관람객에게 부여되는 미션은 2가지로 구성되며, 각 미션의 구체적인 내용은 부스 운영팀이 별도 정의한다. 미션 달성 조건에 따른 보상 구조는 다음과 같다.

| 달성 조건 | 기본 보상 | 추가 보상 (보너스) | 총 수령 $CROSS |
|-----------|----------|-------------------|---------------|
| 미션 1만 달성 | 10 $CROSS | - | **10 $CROSS** |
| 미션 2만 달성 | 10 $CROSS | - | **10 $CROSS** |
| 미션 1 + 2 모두 달성 | 20 $CROSS | +10 $CROSS (보너스) | **30 $CROSS** |

### 2.2 보상 정책 원칙

- 1인당 최대 수령 한도: **30 $CROSS**
- 미션 1과 미션 2를 모두 달성한 경우에만 보너스 10 $CROSS 추가 지급
- 동일 지갑 주소로 중복 수령 불가 (어뷰징 방지)
- 토큰 전송은 반드시 **인포데스크 직원 + NEXUS 직원 이중 확인** 후 진행

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 미션 확인 및 토큰 전송 프로세스

전체 프로세스는 다음 단계로 진행된다.

| Step | 담당자 | 행위 | 설명 |
|------|--------|------|------|
| 1 | 인포데스크 직원 | 미션 달성 여부 확인 | 관람객이 완수한 미션을 확인하고 증빙 자료를 검토한다 |
| 2 | 인포데스크 직원 | 달성 조건 선택 | 미션 1만 / 미션 2만 / 미션 1+2 중 해당하는 조건을 선택한다 |
| 3 | 인포데스크 직원 | $CROSS 토큰 전송 | 조건에 맞는 $CROSS 금액을 관람객 지갑으로 전송한다 |
| 4 | NEXUS 직원 | 어뷰징 여부 확인 | 전송 건에 대해 어뷰징이 아님을 확인하고 체크박스를 체크한다 |
| 5 | 인포데스크 + NEXUS | 체크박스 기입 | 모든 확인 항목을 체크박스 형태로 체크하여 건별 문제 없음을 인증한다 |

### 3.2 건별 전송 기록 필드

각 전송 건에 대해 다음 정보가 기록되어야 한다.

| No. | 필드명 | 설명 | 비고 |
|-----|--------|------|------|
| 1 | 전송 일시 | 전송이 발생한 날짜 및 시간 | YYYY-MM-DD HH:MM (PST) |
| 2 | 관람객 지갑 주소 | 토큰을 수령할 지갑의 온체인 주소 | 중복 검증에 활용 |
| 3 | 달성 미션 유형 | 미션 1 / 미션 2 / 미션 1+2 | 드롭다운 선택 |
| 4 | 배포 $CROSS 수량 | 조건에 따른 자동 계산값 | 10 / 10 / 30 |
| 5 | TX Hash | 블록체인 전송 트랜잭션 해시 | 전송 완료 후 자동 기록 |
| 6 | 인포데스크 직원 확인 | 미션 확인 및 전송 담당 직원의 체크 | 체크박스 (Y/N) |
| 7 | NEXUS 직원 확인 | 어뷰징 방지 확인 담당자의 체크 | 체크박스 (Y/N) |
| 8 | 비고 | 특이사항 기록 (의심 사례 등) | 선택 입력 |

### 3.3 체크박스 기반 검증 프로세스

모든 전송 건은 아래 체크박스가 완료되어야 정상 처리된 것으로 간주된다.

| No. | 체크 항목 | 담당자 | 체크 |
|-----|----------|--------|------|
| 1 | 관람객이 미션을 실제로 완수했는가? | 인포데스크 | ☐ |
| 2 | 미션 달성 유형(미션 1/2/1+2)이 정확히 선택되었는가? | 인포데스크 | ☐ |
| 3 | 수령 지갑 주소가 정확하게 입력되었는가? | 인포데스크 | ☐ |
| 4 | 동일 지갑으로 이미 전송된 기록이 없는가? | NEXUS | ☐ |
| 5 | 전송 금액이 조건별 기준금액과 일치하는가? | NEXUS | ☐ |
| 6 | 어뷰징 의심 사항이 없는가? | NEXUS | ☐ |

### 3.4 어뷰징 방지 규칙

| No. | 규칙명 | 상세 내용 |
|-----|--------|----------|
| 1 | 지갑 주소 중복 검증 | 동일 지갑 주소로 2회 이상 전송을 차단한다. 시스템에서 이미 등록된 지갑 주소를 자동으로 검증하여 중복 전송을 방지한다. |
| 2 | 이중 확인 체계 | 인포데스크 직원의 1차 확인 후, NEXUS 직원이 2차로 어뷰징 여부를 검토한다. 양쪽 모두 확인해야 정상 처리된다. |
| 3 | 의심 건 별도 관리 | 어뷰징이 의심되는 건은 비고란에 사유를 기재하고, 일별 정산 시 별도 검토한다. |
| 4 | 일별 한도 모니터링 | 일별 총 전송 건수 및 총 전송 $CROSS 수량을 모니터링하여 비정상적인 급증을 감지한다. |
| 5 | 전송 시간대 검증 | 부스 운영 시간 외 전송 건은 자동으로 플래그 처리한다. |

## 4. 일별 정산 및 검증 체계

### 4.1 일별 정산 프로세스

행사 기간 3일간 매일 부스 운영 종료 후 일별 정산을 실시한다.

1. 해당일 발생한 모든 전송 건을 집계한다
2. 각 건별로 인포데스크 직원 확인(체크박스) 완료 여부를 검증한다
3. 각 건별로 NEXUS 직원 확인(체크박스) 완료 여부를 검증한다
4. 총 발송 건수 및 총 발송 $CROSS 수량을 집계한다
5. 체크박스 미완료 건, 의심 건에 대해 별도 리뷰한다
6. 정산 담당자가 일별 정산 보고서에 서명한다

## 6. 비기능 요구사항 (Non-Functional Requirements)

### 6.1 보안 요구사항

- 토큰 전송용 지갑의 프라이빗 키는 총괄 책임자만 관리하며, 인포데스크 직원에게는 전송 기능만 제공한다
- 모든 전송 기록은 블록체인 상에서 검증 가능하며 위변조가 불가능하다
- 시스템 접근 권한은 역할별로 분리하여 관리한다

### 6.2 가용성

- 부스 운영 시간 중 99.9% 가용성을 보장한다
- 네트워크 장애 시 오프라인 기록 후 복구 시 일괄 반영 가능해야 한다

### 6.3 사용성

- 인포데스크 직원이 최소한의 교육으로 즉시 사용 가능해야 한다
- 모바일 및 태블릿 환경에서도 정상 동작해야 한다
- 한국어/영어 다국어 지원이 필요하다

## 7. UI/UX 요구사항

### 7.1 전송 관리 화면 (Info Desk View)

1. 관람객 지갑 주소 입력란 (QR 스캔 또는 수동 입력)
2. 미션 달성 유형 선택 드롭다운 (미션 1 / 미션 2 / 미션 1+2)
3. 자동 계산된 배포 $CROSS 수량 표시
4. 체크박스 영역: 미션 확인 완료 / 지갑 주소 확인 / 어뷰징 점검 완료
5. 전송 버튼 (모든 체크박스 완료 시에만 활성화)
6. TX Hash 기록 및 전송 완료 상태 표시

### 7.2 일별 정산 화면 (Settlement View)

1. 일별 전송 목록 전체 조회 (필터: 날짜, 미션 유형, 체크 상태)
2. 집계 정보: 총 건수, 총 $CROSS, 확인 완료율
3. 미완료 건 하이라이트 표시
4. 정산 담당자 서명란 (전자 서명 또는 텍스트 입력)
5. PDF/Excel 내보내기 기능

## 8. 화면 흐름 (Flow)

**전송 화면 Flow:**

```
지갑 주소 입력 → 중복 검증 → 미션 유형 선택 → 금액 자동 계산
→ 체크박스 확인 (인포데스크) → 체크박스 확인 (NEXUS)
→ 전송 실행 → TX Hash 기록 → 완료
```

**정산 화면 Flow:**

```
날짜 선택 → 전송 목록 조회 → 건별 체크박스 검증 → 집계 확인
→ 미완료 건 리뷰 → 정산 담당자 서명 → 보고서 내보내기
```
