# 텔레그램 봇 TASKS.md 동기화 구축

## 배경
Cowork 스케줄링 태스크와 텔레그램 봇 태스크가 불일치하는 문제 발견. 텔레그램 봇이 `data/tasks.json`을 독립적으로 관리하고 있어 Obsidian TASKS.md와 전혀 연동이 안 된 상태였음.

## 해결 아키텍처

```
맥북 TASKS.md (Obsidian)
    ↕ sync_tasks.sh (cron 5분마다)
GitHub Gist (Public, ID: a8587b66f6e2b36d85f00d57a9003080)
    ↕ Railway bot.py (Gist read/write)
텔레그램 봇
```

## 주요 작업 내역

### 1. GitHub Gist 설정
- TASKS.md 내용을 Public Gist로 생성
- Gist ID: `a8587b66f6e2b36d85f00d57a9003080`
- Public으로 설정해야 Railway에서 토큰 없이 읽기 가능 (GitHub 보안 정책상 코드에 토큰 push 시 자동 revoke됨)

### 2. 맥북 환경변수 설정 (~/.zshrc)
```bash
export GITHUB_TOKEN="..."   # gist + repo 권한
export GIST_ID="a8587b66f6e2b36d85f00d57a9003080"
```

### 3. sync_tasks.sh 설치
- 경로: `~/Desktop/5. 넥써쓰_Claude/sync_tasks.sh`
- 기능: push(로컬→Gist), pull(Gist→로컬), sync(수정시간 비교 후 자동 판단)
- 백업: `~/.tasks_backup/` (최대 30개)

### 4. cron 등록 (5분마다 자동 sync)
```
*/5 * * * * /Users/baht/Desktop/5. 넥써쓰_Claude/sync_tasks.sh sync >> /tmp/tasks_sync.log 2>&1
```

### 5. Railway 봇 코드 교체
- 기존: `data/tasks.json` 독립 관리
- 변경: GitHub Gist TASKS.md 직접 read/write
- 배포 경로: `~/Desktop/5. 넥써쓰_Claude/넥써쓰_Vault/5. 자료/telegram-claude-bot/claude-telegram-bot/bot.py`
- GitHub repo: `https://github.com/baht-mania/claude-telegram-bot`

### 6. Railway 환경변수
- `GITHUB_TOKEN`: GitHub PAT (gist + repo 권한)
- `GIST_ID`: `a8587b66f6e2b36d85f00d57a9003080`
- Railway CLI로 설정: `railway variables set GITHUB_TOKEN="..."`

## 트러블슈팅 기록

### GitHub 토큰 자동 revoke 문제
- 원인: bot.py에 토큰 하드코딩 후 git push → GitHub 보안 스캔이 자동 감지 및 revoke
- 해결: Gist를 Public으로 변경 (읽기는 토큰 불필요), 쓰기용 토큰은 Railway Variables에만 저장

### Railway Variables 보라색 표시 문제
- 원인: Variables 저장 후 Deploy가 트리거되지 않음
- 해결: Railway CLI (`railway variables set`)로 직접 설정

### bot.py syntax error 반복
- 원인: nano 붙여넣기 시 줄바꿈 문자 오염
- 해결: 최종적으로 clean bot.py 파일을 새로 작성해서 덮어씀
