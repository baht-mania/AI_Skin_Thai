**요구사항 확정:**
- 플랫폼: 웹 앱 (내부 팀 사용)
- AI: Claude API 활용
- 입출력: Excel / CSV 포맷
- 다국어: 글로벌 다중 언어 지원
- 콘텐츠 유형: UI 텍스트, 스토리, 아이템명, 시스템 메시지 등 전체

### 5단계 — 구현 (Phase 1 ~ 10)

#### Phase 1 — 프로젝트 초기화
- **Next.js 16** 프로젝트 생성
- **Prisma 7** ORM + **SQLite** 데이터베이스 구성
- **shadcn/ui** 컴포넌트 라이브러리 설치
- **Noto Sans KR** 폰트 적용 (한국어 최적화)
- 기본 레이아웃 및 라우팅 구조 확립
- Prisma 스키마: 8개 테이블 정의

#### Phase 2 — 프로젝트 관리
- 프로젝트 목록 페이지 (카드 그리드)
- 프로젝트 생성/수정/삭제 (CRUD)
- 프로젝트 대시보드 (번역 진행률, 통계)
- 사이드바 네비게이션 레이아웃

#### Phase 3 — 용어집 관리
- 용어집 테이블 (TanStack Table)
- 용어 추가/수정/삭제
- Excel 가져오기 / 내보내기
- 프로젝트별 용어집 분리

#### Phase 4 — 파일 업로드
- Excel (.xlsx) / CSV 파일 파싱
- 컬럼 매핑 UI (소스 컬럼 선택)
- 번역 항목 DB 저장

#### Phase 5 — 번역 엔진
- **Claude API** (claude-sonnet-4-5) 연동
- 콘텐츠 유형별 특화 프롬프트
  - UI 텍스트, 스토리/내러티브, 아이템명, 시스템 메시지
- 용어집 자동 주입
- 배치 처리 (대량 번역)
- 번역 진행 상태 실시간 표시

#### Phase 6 — 번역 메모리 (TM)
- TM 데이터베이스 구축
- **퍼지 매칭** (Levenshtein 알고리즘)
- 유사도 임계값 설정 가능
- TM 조회/추가/수정/삭제

#### Phase 7 — 검토 시스템
- 원문 / 번역문 나란히 비교 뷰
- 인라인 번역 편집
- 승인 처리 → 자동 TM 저장
- 전체/미검토/승인/거부 필터

#### Phase 8 — QA 엔진
6개 검사 모듈:
1. **플레이스홀더 검사** — {변수} 누락/불일치
2. **길이 검사** — 원문 대비 번역 길이 비율
3. **용어집 검사** — 지정 용어 미사용
4. **누락 번역 검사** — 미번역 항목
5. **서식 검사** — HTML 태그, 마크업 일치
6. **숫자 검사** — 숫자 누락/변경

#### Phase 9 — 내보내기 & 설정
- Excel / CSV 다운로드
- 원문+번역문 컬럼 구조로 출력
- 설정 페이지: API 키, 기본 언어, UI 환경설정

#### Phase 10 — 버그 수정 및 최종 검증
- 사이드바 레이아웃 겹침 버그 수정
- 전체 페이지 dev server 프리뷰로 동작 확인
- 모든 기능 정상 동작 검증 완료
