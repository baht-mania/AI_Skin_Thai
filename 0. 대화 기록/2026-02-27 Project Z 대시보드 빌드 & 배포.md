### 2단계 — 빌드 에러 수정

**발견된 에러와 해결:**

1. **Prisma 모듈 경로 오류** — Turbopack이 커스텀 출력 경로를 지원하지 않음
   - `provider = "prisma-client"` → `"prisma-client-js"`로 변경
   - 임포트: `@/generated/prisma` → `@prisma/client`

2. **빌드 시 DB 연결 시도** — Next.js 정적 생성이 DB 연결을 요구
   - 59개 파일에 `export const dynamic = 'force-dynamic'` 추가
   - PrismaClient를 Proxy로 감싸 lazy initialization 구현

3. **Prisma v7 `$use()` 디프리케이션** — `$extends()` 패턴으로 재작성

4. **Supabase 미설정 시 크래시** — `isSupabaseConfigured()` 가드 추가
   - `middleware.ts`, `server.ts`, `layout.tsx`에 null 체크

5. **Recharts 타입 에러** — Tooltip formatter 타입 캐스트

6. **react-day-picker captionLayout** — `"dropdown-buttons"` → `"dropdown"`

### 4단계 — Vercel 배포

**발생한 문제들과 해결:**

1. **`supabase` null 체크 누락** — `auth/callback/route.ts`에 null 가드 추가
2. **Resend API 키 모듈 로드 에러** — `new Resend()`를 모듈 최상위에서 함수 내부로 이동
3. **Prisma `directUrl` 미지원** — `prisma.config.ts`에서 제거
4. **환경변수 빌드 미포함** — Vercel `NEXT_PUBLIC_` 변수는 빌드 시점에 포함됨. "Use existing Build Cache" 해제 후 재배포 필요
5. **API 키 잘림** — Vercel에 붙여넣기 시 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 값이 잘려 저장됨 → 재입력으로 해결

## 주요 기술적 결정

| 결정 사항 | 이유 |
|-----------|------|
| Prisma v7 `$extends()` 사용 | `$use()` 미들웨어가 v7에서 디프리케이트됨 |
| PrismaClient Proxy 래핑 | 빌드 시점 DB 연결 방지 (lazy init) |
| `force-dynamic` 전체 적용 | Next.js 정적 생성이 DB 연결을 요구하는 문제 해결 |
| `isSupabaseConfigured()` 가드 | Supabase 미설정 환경에서 graceful degradation |
| SQL Editor 직접 실행 | 로컬→Supabase DB 직접 연결 불가 (IPv4/pooler 이슈) |
