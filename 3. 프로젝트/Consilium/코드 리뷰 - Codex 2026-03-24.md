# Consilium 코드 리뷰 — Codex (2026-03-24)

## 요약
Codex가 Consilium(구 MiroFish) 코드베이스를 리뷰한 결과. 5개 이슈 발견.

---

## Findings

### P0 — 서버 측 인증 없음
- 프론트 라우터는 Supabase 로그인 보호가 있지만, Flask 백엔드 API는 인증 없이 누구나 호출 가능
- 프로젝트 조회/삭제/리셋, 보고서 삭제/다운로드, 시뮬레이션 시작/중지 모두 공개
- **파일**: `backend/app/__init__.py`
- **조치**: API_SECRET_TOKEN Bearer 토큰 가드 추가 (임시 방어) → 상용화 전 Supabase JWT 검증 필수

### P0 — 크레딧이 localStorage만으로 관리
- 크레딧 잔액/차감/충전이 전부 클라이언트 localStorage에서만 동작
- 백엔드 실행 API에 크레딧 검증 없음 → 직접 API 호출 시 무상 사용 가능
- **파일**: `frontend/src/composables/useCredits.js`
- **조치**: 상용화 시 서버사이드 크레딧 차감 필수

### P1 — 라운드 30 하드코딩
- 가격 UI가 항상 `:rounds="30"`으로 계산, 실제 슬라이더 값과 무관
- 슬라이더가 30이 아닐 때 과금 표시와 실제 실행이 어긋남
- **파일**: `frontend/src/views/Home.vue`
- **조치**: ✅ 수정 완료 (슬라이더 실제 값 연동)

### P1 — TaskManager 인메모리 유실
- `TaskManager`가 메모리 singleton으로만 상태 저장
- 배포/재기동/슬립 이후 그래프 빌드, 보고서 생성 진행 조회 끊김
- 502/History 빈 데이터의 근본 원인
- **파일**: `backend/app/models/task.py`
- **조치**: 파일 기반 또는 Supabase 영속화 필요 (다음 스프린트)

### P2 — traceback 클라이언트 노출
- `traceback.format_exc()`가 JSON 응답에 포함되어 내부 경로/구현 정보 노출
- **파일**: `backend/app/api/graph.py`, `report.py`, `simulation.py`
- **조치**: ✅ 수정 완료 (서버 로그에만 기록, 클라이언트에 generic 에러 반환)
